Vim Notes, shortcuts, and more

Vim Stuff

:w other_file

Use to save the open file with a different file name

v gg G J

Use to minify the file. Basically: Visual mode, goto top, select to bottom, minify

:tabnew newFile.html

Then use gT to cycle between the tabs. Turns out the same keybinds made their way to the VS Code plugin, which is much nicer than the defaults.

Motivation

Basically, I'm trying to get myself to use VIM as my main editor, rather than the VS Code vim plugin. Most of the functionality can be pretty easily replicated, and it's incredibly performant. I've run into the issue, though, that copying and pasting text across open VIM instances is a problem, specifically with panes/splits. It turns out it works fine with tabs. Just:

I also learned about xTerm 256 colors today and tweaked my VIM theme, originally based on oneDark. Some of the yellows and greys were hard to see, so I adjusted the colors to make the contrast better.

Extensions

Bracey: use for live-server. It highlights the elements as you navigate too, which is helpful.

Airline: visual feedback of modes as you switch.

NerdTree: use for a visual file tree.

Emmet-Vim: use for emmet shortcuts. It's a bit cumbersome with the default keybindings, but it works

YouCompleteMe: autocompletion, makes it more competitive with VSCode as an actual workspace. Vim has built in ways to do this, but they're less convienent. It is a bit of work to install though.

Vim-Prettier: has 135 npm security vulnerabilities as of 6-30-21.
